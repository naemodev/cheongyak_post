# 청약우체부 📮

성남·용인·위례·하남·광주(경기)의 분양·임대 청약 공고를 하루 5번 확인해서 새 소식을 텔레그램으로 보내주는 서비스. 대시보드로 한눈에 볼 수도 있다.

- **출처**: 청약홈(민영·공공 APT / 무순위·잔여 / 공공지원임대) + LH(분양 / 임대 / 신혼희망타운)
- **알림**: 📢 모집공고 게시 / ✅ 청약접수 시작 / ⏰ 접수 마감 임박 (중복 발송 안 함)
- **실행**: GitHub Actions cron — KST 08·11·14·17·20시 자동
- **대시보드**: GitHub Pages — `https://naemodev.github.io/cheongyak_post/`

## 동작 방식

```
청약홈 API(날짜기반) ┐
                    ├ 합치기 → 지역 필터 ┬ 트리거 → 중복 제외 → 텔레그램 발송
LH API(상태기반)    ┘                   └ 상태·자격·태그 계산 → docs/data/listings.json (대시보드)
```

- 청약홈은 날짜로, LH는 공고상태로 트리거를 판정한다.
- 중복은 `(출처+공고+트리거)` 단위로 `state/notified.json`에 기록해 막는다.

## 손볼 때 (전부 한 곳씩)

- **지역·수집 유형·트리거 기준** → `config.js`
- **자격 기준 / 태그(persona)** → `src/eligibility.js` (소득·자산 기준값은 해마다 바뀌므로 연 1회 `BASE_YEAR`와 함께 갱신)
- **실행 시각** → `.github/workflows/notify.yml`의 cron

## 자주 하는 것

- **알림을 다시 받아보기**: `state/notified.json`을 `{"updatedAt":null,"sent":[]}`로 비우고 → Actions에서 seed 끄고 Run
- **코드 고쳐 푸시할 때**: Action이 원격에 자동 커밋하므로 항상 `git pull --rebase origin main` 먼저 → `git push`

## 참고

- **LH 건은 주소·세대수·마감일이 비어 있다.** LH 목록 API가 그 값을 안 주고 공고상태만 주기 때문 (버그 아님). 더 정확히 보려면 각 공고문에서 확인.
- 자격 요약은 **유형별 일반기준(참고용)**. 실제 신청 자격은 공고문이 정답.
- 대시보드 메모는 **그 브라우저(기기)에만** 저장된다.
